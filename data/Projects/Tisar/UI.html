Tisar: registration flow

Screen 1

  • Registration
    • Registration flow: Admin Panel
      - Webpage admin panel: admin registers users and provides them with one and only password
      Scenario : Creation of a new user, creds in the mobile app:
      When : user is registered from the admin panel http://www.tisar.com/
      And : admin created a user by specific username
      And : admin assigned a permanent password to the user
      Then : user can register under the provided credentials
      And : created user credentials is not possible to change from within the mobile application.

    • User-story: Login Flow
      When: user logs in
      Then: validation takes place by user name, and by password
      And: validation error messages should be generated:
      And: the input fields are highlighted in red
      NB! Validation Errors are to be provided by Aviv!
