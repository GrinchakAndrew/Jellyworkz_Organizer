Inside a factory

  		•

    • Sub-factory component clickable (inside a factory):
      • General Description :
        Component : Sub-factory component
        - Searching by status of machines or by name or by serial number
        - Search-filtering machines -> next component
        - Dropdown date: month, years
        Scenario: Sorting by Date Selection (should be only months, not a full date)
        Given : user applied the Date Filter
        And : the selection of a month and year took place to go back in time
        When : the selection dropdown is set to new date (month and a year)
        Then : the filtering takes place
        And : the 'Inside a Factory' component is refreshed to the filtered view

        Scenario: Sorting by Machine Status (Filter only by Broken Machines)
        Given : user applied the Filter by Machine Status
        When : the selection dropdown is set to the status of a Machine, e.g., 'Broken'
        Then : the filtering takes place
        And : the 'Inside a Factory' component is refreshed to the filtered view

        Scenario : Sort-filtering by All Machines
        Given : user applied the Filter by 'All Machines'
        When : the corresponding option on the header has been click-selected
        Then : the filtering takes place
        And : the 'Inside a Factory' component is refreshed to the filtered view: All of the Machines under the given Factory populate the component's view.

        Scenario: Editing status of a machine:
        NB! IT IS NOT POSSIBLE TO CHANGE THE STATUS OF MACHINES FROM THE APPLICATION, ONLY FROM THE WEBSITE (ADMIN PANEL)
        NB! Editing status of a machine: only possible while the month is not over yet.
        Given : the targeted machine is viewed within the current monthly
        And : the machine is not viewed under the 'Notification' option
        When : Admin Logs in to Edit machine status from the Admin Panel
        Then : all of the changes are brought up to date inside the application
